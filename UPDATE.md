# PLATO v0.10.0 — Платон Сергеевич 2.0

Агент перестроен с «чата над JSON» на tool-using архитектуру.

## Инструменты
- `explain_metric` — точное объяснение LLCR, расходов, выручки, CAPEX, прибыли, себестоимости, финансирования и ТЭП.
- `trace_metric` — трассировка происхождения показателя.
- `goal_seek` — универсальный аналог Excel «Подбор параметра».
- `simulate_change` — сценарный пересчёт на копии модели.
- `find_anomalies` — структурные контроли модели и ГлавАПУ.
- `get_methodology` — встроенные утверждённые правила PLATO.

## Goal Seek
Поддерживаются изменяемые параметры:
- цена покупки;
- ставка основного строительства;
- стартовая цена квартир;
- цена коммерции;
- цена паркинга;
- социальная компенсация;
- спред БРИДЖ.

Целевые показатели:
- LLCR;
- маржа;
- чистая прибыль;
- NPV;
- IRR equity.

Пример:
«За сколько максимум можно купить проект, чтобы LLCR был не ниже 1,20?»

Платон обязан вызвать Goal Seek, а не оценивать ответ самостоятельно.

Для многоочередного проекта банковская логика по умолчанию ориентирована на слабейшую очередь.

## Важно
Все инструменты read-only: пересчёт выполняется на копии состояния модели.
Финансовые формулы основной модели этой версией не менялись.

Текущая многоочередная модель всё ещё использует фазовый финансовый движок; отдельная единая банковская facility общего Bridge с формальным межочередным рефинансированием пока не реализована.
